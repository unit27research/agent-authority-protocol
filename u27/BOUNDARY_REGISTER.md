## Public Claims

| Claim | Boundary |
|---|---|
| Agent Authority Protocol is an open proposal for classifying delegated authority in AI agents before deployment. | Proposal status only. Not an official standard. |
| The protocol uses the Authority-Reach-Delegation model, or ARD. | ARD is the model used by this package. |
| ARD classifies authority, reach, and delegation rights. | Classification aid only. Does not enforce runtime authority. |
| The package includes class tables, examples, a dossier template, evidence requirements, and schema examples. | Limited to repository contents. |
| The protocol can help scope review before agents receive tools, credentials, autonomy, or delegation rights. | Review aid only. Does not replace security, legal, compliance, or authorization review. |

## Explicit Non-Claims

- Not an official standard.
- Not a certification system.
- Not legal, compliance, security, audit, procurement, or professional advice.
- Not an IAM implementation.
- Not an authorization protocol.
- Not runtime monitoring.
- Not proof that any agent is safe to deploy.
- Not market validation.

## Review Notes

The authority classifications are core to the package. The project should preserve the A0-A6 ladder as a first-class public surface, especially the distinction between autonomous operation and delegating authority.
