# Claim Boundaries

Agent Authority Protocol should be described as a public protocol package for Authority-Reach-Delegation classification.

It should not be described as an official standard, a security product, a compliance framework, or proof that an AI agent is safe to deploy.

## Safe Claims

Safe public claims:

- Agent Authority Protocol is an open proposal for classifying delegated authority in AI agents before deployment.
- The protocol uses the Authority-Reach-Delegation model, or ARD.
- ARD classifies authority, reach, and delegation rights.
- The protocol provides class tables, example classifications, a dossier template, evidence requirements, and schema examples.
- The protocol can help teams scope review before agents receive tools, credentials, autonomy, or delegation rights.
- The protocol is intended to complement identity, access, security, AI governance, and review frameworks.

## Claims That Need Evidence

These claims require matching evidence before publication:

- The protocol has been adopted by external teams.
- The protocol improves agent safety outcomes.
- The protocol reduces incidents.
- The protocol maps cleanly to a specific compliance requirement.
- A specific organization can use the protocol without modification.
- A specific implementation conforms to the protocol.

## Unsafe Claims

Do not claim:

- This is the first AI agent authority standard.
- This is an official standard.
- This replaces IAM, authorization systems, runtime monitoring, OWASP guidance, NIST AI RMF, ISO/IEC 42001, SOC 2, legal review, compliance review, or security review.
- This makes agent deployment safe.
- This certifies an AI agent.
- This proves an agent is authorized.
- This solves agent governance.
- This is enterprise-ready or market-validated.

## Public Framing

Preferred public framing:

> Agent Authority Protocol is a Unit27 public protocol package for Authority-Reach-Delegation classification before AI agents receive tools, credentials, autonomy, or delegation rights.

## Standing Boundary

Agent Authority Protocol is a classification and review aid.

It does not grant authority, enforce authority, revoke authority, authenticate agents, authorize API calls, or certify deployment readiness.
